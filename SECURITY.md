# Security Policy

Open Storyboard Canvas 是本地优先的桌面应用，但它会在用户触发 AI 生成、编辑、全景或自定义供应商请求时，把提示词、参考图和参数发送给用户选择的供应商或本地工具链。请按下面的边界理解和报告安全问题。

## 支持范围

安全修复优先覆盖：

- `main` 分支
- 最新 GitHub Release

历史版本可能不再单独补丁发布。升级到最新版本通常是首选修复方式。

## 报告漏洞

如果你发现会泄露 API Key、绕过本地权限、破坏项目数据、执行非预期命令或导致供应商请求被篡改的问题，请优先使用 GitHub Security Advisory 私下报告。

如果仓库尚未开启 Security Advisory，请先通过 GitHub issue 提交最小化描述，不要公开可直接利用的细节、真实密钥、私有接口、用户数据或完整项目文件。维护者确认沟通渠道后，再补充复现材料。

报告时请包含：

- 受影响版本或 commit。
- 操作系统和运行方式。
- 复现步骤。
- 预期影响。
- 你已经确认不含敏感信息的日志或截图。

## API Key 与本地凭据

- API Key 和自定义供应商配置由应用本地设置存储保存，不应提交到 Git。
- 不要把真实密钥写进 README、issue、PR、测试 fixture、截图、录屏或日志。
- Dreamina / 即梦路线依赖本地 CLI 登录态时，请把 CLI 登录态视为本机敏感凭据。
- 如果密钥疑似泄露，请先在对应供应商后台吊销或轮换，再清理本地配置和相关日志。

## 数据边界

- 画布项目、节点、边、历史记录和图片引用保存在本机 Tauri 应用数据目录。
- 应用不提供云同步账号系统；备份、迁移和删除本地数据由用户自行管理。
- 生成请求会把所需提示词、参考图和参数发送到所选供应商或本地工具链。供应商侧的数据留存、审核、训练使用和合规责任以其条款为准。
- 公开 issue 或 PR 前，请删除项目文件、截图和日志中的个人路径、客户资料、未公开素材和其他敏感信息。

## 依赖与供应商

第三方 npm/Rust 依赖、AI 供应商、聚合器、代理服务和本地 CLI 都有各自的安全模型。本项目会尽量避免记录密钥和大型敏感载荷，但不能替代供应商安全审计、账号风控或组织级数据合规审查。
